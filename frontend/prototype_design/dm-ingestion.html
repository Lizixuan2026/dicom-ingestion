数据接入 / Ingestion

将本地文件、文件夹或 Manifest 接入平台，生成 Dataset、初始 Data Version 和 Manifest。

不同接入类型使用不同解析逻辑：医学影像解析 DICOM / 医学元数据，通用数据解析字段结构，多模态数据校验模态配对关系。

通用单模态数据用于接入文本、文档、图片、音频、视频、表格、CSV、JSON 等单一模态数据。系统第一版只做基础格式校验，不自动识别 Prompt、Response、Candidate、Label 等业务字段。

多模态数据用于接入图文、影像报告、音文、视频文本等组合数据。系统第一版不理解具体任务字段，只校验模态组合和样本引用关系。

多模态结构说明

多模态数据需要通过 Manifest 或文件结构描述不同模态之间的关系。第一版系统只做基础引用关系校验，不自动理解图文、音文或视频文本语义。

上传后系统将校验：

  • Manifest 是否可读取
  • 每条样本是否有 sample_id
  • 每条样本是否包含声明的模态
  • 文件引用路径是否可访问
  • 是否存在缺失模态样本
  • 是否存在重复 sample_id
  • 文件大小和记录数统计

复杂字段映射将在后续标注项目创建或数据处理阶段按任务类型配置。

多模态 Manifest 规范

Manifest 需要声明 sample_id、模态组合和每个模态的文件引用路径，第一版系统只做基础引用关系校验。

字段说明
sample_id样本唯一标识，必须存在且不可重复。
modalities声明样本包含的模态，例如 IMAGE、TEXT、DICOM。
files各模态对应的本地相对路径或 Manifest 内引用路径。
metadata可选字段，保存来源、标签或业务补充信息。
